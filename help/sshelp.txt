Option
-#
ファイルまたはプロジェクトについて、History コマンドで表示するエントリ数を指定します。
  -#
    ファイルまたはプロジェクトのすべてのエントリを指定します。
  -#<数値>
    表示するファイルまたはプロジェクトのエントリ数を指定します。

-B
Properties、Add、および Filetype の各コマンドでは、ファイルがバイナリであるかテキストであるかを指定します。History および Diff コマンドでは、-B オプションは簡略形式を指定します。Merge コマンドでは、このオプションはマージ先プロジェクトの基本バージョンを指定します。
  -B
    Properties、Add、および Filetype コマンドに対してバイナリ ファイルを指定します。History コマンドと Diff コマンドについては、簡略形式を指定します。
  -B-
    Properties、Add、および Filetype コマンドに対してテキスト ファイルを指定します。
  -B<番号>
    Merge コマンドで使用する場合は、マージ先のプロジェクトの基本バージョンを指定します。
  -Bd<日付>
    Merge コマンドでは、示された日付/時間スタンプを持つ基本バージョンを指定します。
  -Bl<ラベル>
    Merge コマンドでは、示されたラベルを持つ基本バージョンを指定します。

-C
コマンドにコメントを指定します。-C オプションを指定しないと、Visual SourceSafe は、コマンド ラインで指定した項目ごとにコメントを要求します。このオプションを指定すると、すべての項目が同じコメントを受け取ります。
  -C
    すべての項目が同じコメントを受け取るように指定します。
  -C-
    コマンドにコメントを使用しないように指定します。
  -C<text>
    すべての項目のコメントとして文字列テキストを使用します。
  -C@<ファイル名>
    指定したファイルの内容を、すべての項目のコメントとして使用します。
  -C?
    Visual SourceSafe の既定の動作に戻り、個々のファイルごとにコメントを要求します。

-D
Visual SourceSafe における相違点の表示方法を制御します。このオプションは、処理速度を短縮しディスク容量を節約するために、バイナリ ファイルでよく使用されます。
  -D
    相違点の表示に使用する形式を指定します。この表示形式では、追加、削除、変更、および最適化された行の一覧が、見やすく表示されます。
  -D-
    Visual SourceSafe データベースにあるファイルの最新バージョン以外をすべて表示します。
  -DS<行幅>
    標準の Visual SourceSafe 表示形式を指定します。既定では、Visual SourceSafe はこの表示形式を使用します。行幅は画面に合うように計算され、前後の行は表示されません。たとえば、挿入された行は次のように表示されます。
    27 Ins: hello, world
  -DU<行幅>
    UNIX スタイルの表示形式を指定します。行幅は 70 文字で、前後の 5 行が表示されます。この表示では、UNIX の diff ユーティリティで使用される標準形式に似た形で情報が表示されます。次に例を示します。
    26a27
    > hello, world
  -DV<行幅>
    ビジュアル表示を指定します。ファイル全体が表示され、変更箇所に注釈が付きます。この表示形式では、ファイルの相違点が横に並べて表示されます。
  -DX
    各変更箇所の前後 3 行を表示することを指定します。
  -DX-
    変更箇所の前後の行を表示しないことを指定します。
  -DX<数値>
    変更箇所の前後に <数値> で指定した行数を表示することを指定します。

-E
ファイルのチェックアウト情報など、Dir コマンドの拡張表示を生成します。Share コマンドでは、このオプションは共有ではなく、共有および分岐の操作を指定します。Add、Properties、および Filetype コマンドでこのオプションを使用すると、ファイルのエンコーディングを指定します。
  -E
    Dir コマンドの場合のみファイルを表示します。このオプションは Share コマンドに対しては共有と分岐の操作を指定します。Add、Properties、および Filetype コマンドと共に使用した場合、-E はファイルのエンコーディングを指定します。

-F
ファイルだけを表示し、プロジェクトは表示しません。
  -F
    ファイルの表示を有効にします。
  -F-
    ファイルの表示を無効にします。

-G
取得した作業コピーに対するオプションを設定します。このオプションの多くのスイッチにより、Ss.ini ファイルで設定できるコマンド固有のオーバーライドを実現できます。
  -G
    ローカル バージョンの取得を指定します。
  -G-
    取得操作を指定しません。
  -GCC
    ファイルの内容を比較することにより、ファイルのローカル コピーが最新であるかどうかを判断します。
  -GCD
    日付と時刻を比較して、ファイルのローカル コピーが最新であるかどうかを判断するように指定します。
  -GCK
    チェックサムを使用して、ファイルのローカル コピーが最新であるかどうかを判断するように指定します。
  -GF
    該当するコマンドで Force_Dir 初期化変数を有効にします。
  -GF-
    該当するコマンドで Force_Dir 初期化変数を無効にします。
  -GL<パス>
    現在のフォルダや作業フォルダではなく、指定したフォルダにファイルをコピーします。
  -GN
    テキスト ファイルの行末文字としてラインフィードを指定します。
  -GR
    テキスト ファイルの行末文字として復帰を指定します。
  -GRN
    テキスト ファイルの行末制御文字として復帰とラインフィードのペアを指定します。
  -GTC
    作業コピーに現在の日付と時刻を割り当てるように指定します。
  -GTM
    現在の日付と時刻ではなく、ファイルが最後に変更された日付と時刻をローカル コピーに割り当てるように指定します。
  -GTU
    現在の日付と時刻ではなく、ファイルが最後にチェックインされた日付と時刻をローカル コピーに割り当てるように指定します。
  -GWA
    取得操作とチェックアウト操作で、書き込み専用ファイルの置換、スキップ、またはマージのいずれかを選択するよう求めるダイアログ ボックスを表示します。
  -GWM
    特定の操作 (取得およびチェックアウト) で書き込み専用ファイルをマージします。
  -GWR
    特定の操作 (取得およびチェックアウト) で書き込み専用ファイルを置換します。
  -GWS
    特定の操作 (取得およびチェックアウト) で書き込み専用ファイルをスキップします。

-H
選択したコマンドのヘルプを表示します。このオプションは -? オプションの代わりに使用されます。
  -H
    オンライン ヘルプを表示します。

-I
2 つのファイルを比較する場合に、Visual SourceSafe が無視する要素を指定します。
  -I
    選択した比較要素を無視します。
  -I-
    すべてを無視し、どのような状況でも入力を要求しないようにコマンドに指示します。
  -I-Y
    Yes/No 型質問に対して Yes を回答します。
  -I-N
    Yes/No 型質問に対して No を回答します。
  -IC
    大文字と小文字を無視します。たとえば、大文字の A は小文字の a と同じと見なされます。
  -IE
    行末文字を無視し、その他の違いを変更として報告します。行末文字はオペレーティング システムによって違いがあるため、すべての行について変更として報告しません。-IE は、既定の設定です。
  -IS
    小さな変更を無視します。このオプションを設定すると、実際には一致していないのに、大きな変更セクションが一致しているように見えるのを防ぐことができます。
  -IW
    スペースやタブなどの空白を無視します。機能文字の挿入または削除だけを報告します。

-K
ファイルをチェックインした後も、そのファイルをチェックアウトしたままの状態にするかどうかを指定します。
  -K
    ファイルをチェックアウトしたままにします。
  -K-
    ファイルをチェックアウトしたままにしません。

-L
多くのコマンドで、ファイルまたはプロジェクトのラベルを指定します。Checkout コマンドでこのオプションを使用すると、ローカル バージョンのチェックアウト機能を有効または無効にできます。Merge コマンドでこのオプションを使用すると、マージがローカルで実行され、結果が作業フォルダにチェックアウトされたままになります。
  -L
    ラベルを持つファイルまたはプロジェクトのバージョンを表示します。このオプションを Checkout コマンドで使用すると、ローカル バージョンのチェックアウトが有効になります。Merge コマンドでこのオプションを使用すると、マージがローカルで実行され、結果が作業フォルダにチェックアウトされたままになります。
  -L<名前>
    指定したラベルを持つファイルまたはプロジェクトのバージョンを表示します。
  -L-
    Checkout コマンドで使用すると、ローカル バージョンのチェックアウトが無効になり、現在のデータベース バージョンのチェックアウトが有効になります。

-M
指定したファイルの排他的チェックアウトを有効にします。
  -M
    排他的チェックアウトを無効にします。
  -M-
    個別のファイルの排他的チェックアウトを有効にします。

-N
コマンドに適用するファイルの名前付け規則を指定します。
  -N
    初期化ファイル内の Long_Filenames 初期化変数の設定を使用します。既定では、このオプションは -NL と同じです。
  -NL
    長い名前の名前付けを指定します。
  -NS
    短い名前の名前付けを指定します。

-O
大量の情報が表示される可能性のあるコマンドの出力を制御します。-O を指定しない場合、情報表示に伴って画面が高速にスクロールされ、読むことができないことがあります。
  -O
    すべての出力を表示します。
  -O-
    エラー出力のみを表示します。
  -O@<file name>
    指定したファイルにすべての出力を送信します。履歴レポートなどのレポートをファイルやプリンタにエクスポートする場合は、このオプションを使用します。指定した名前のファイルが既に存在する場合、出力はそのファイルに追加されます。
  "-O&<file name>"
    コマンド出力に加え、標準エラー出力を指定したファイルに送信します。引用符 (") は Windows NT プラットフォームでは必須ですが、他のプラットフォームでは省略可能です。指定した名前のファイルが既に存在する場合、出力はそのファイルに追加されます。
  -O&-
    出力を行いません。

-P
コマンドの対象とするプロジェクトを指定します。
  -P
    カレント プロジェクトを指定します。
  -P<プロジェクト名>
    プロジェクトを名前によって指定します。

-Q
コマンド ラインのコマンドからの画面出力を非表示にします。※Copyコマンド参照
  -Q
    画面出力の中止を指定します。

-R
プロジェクトに対するコマンドをサブプロジェクトに対して再帰的に実行します。ファイル名またはマスクを指定している場合、コマンドはこのオプションを無視します。
  -R
    再帰的操作を指定します。

-S
特定のコマンドで、Smart_Mode 初期化変数をオーバーライドします。
  -S
    スマート モードを有効にします。
  -S-
    スマート モードを無効にします。

-U
ファイルまたはプロジェクトについてのユーザー情報を表示します。
  -U
    ファイルまたはプロジェクトの現在のユーザーを指定します。
  -U<名前>
    <名前>のユーザーを指定します。

-V
ファイルまたはプロジェクト (項目) のバージョンを示します。
  -V<番号>
    示されたバージョン番号のバージョンを指定します。
  -Vd<M-DD-YY[;hh:mm]>
    示された日付時刻スタンプを持つバージョンを指定します。
  -Vl<ラベル>
    示されたラベルを持つバージョンを指定します。

  項目の後にセミコロンとバージョン番号を続けて指定することによって、特定の項目のバージョン番号を指定できます。また、このオプションを使用してバージョン範囲を指定することもできます (必要な最終バージョン以前)。

  バージョン 14 からバージョン 1 にさかのぼってバージョンを表示する。
    ss History -V14
  バージョン 5、4、および 3 を表示する。
    ss History -V5~3
  日付が 1992 年 2 月 29 日のバージョンを表示する。
    ss History -Vd2-29-92
  Final Beta というラベルを持つバージョンを表示する。
    ss History -VlFinal Beta
  1995 年 3 月 3 日の午前 9 時から午後 3 時までの $/Test プロジェクトのバージョンを表示する。
    ss History $/test -Vd3/03/95;3:00p~3/03/95;9:00a
  Help.c のバージョン 6 を表示する。
    ss History -VHelp.c;6
  Show.prg を 1992 年 2 月 29 日に表示されたとおりに表示する。
    ss History -VShow.prg;d2-29-92 

-W
作業用コピーを読み書き可能にするか、読み取り専用にするかを指定します。既定では、ファイルのチェックアウト時に、作業コピーが変更できないように読み取り専用になっています。Ss.ini ファイルの Use_ReadOnly 初期化変数により、この既定の動作を変更できます。 
  -W
    作業用コピーを読み書き可能に設定します。
  -W-
    作業用コピーを読み取り専用に設定します。

-Y
ユーザー名、またはユーザー名とパスワードを指定します。このオプションは、別のユーザーとしてコマンドを実行する場合に使用します。
  -Y<ユーザー名>
    ユーザー名を指定します。
  -Y<ユーザー名>,<パスワード>
    ユーザー名とパスワードを指定します。

-?
選択したコマンドのヘルプを表示します。オンライン ヘルプを表示するには、-H オプションを使用することもできます。


Command

About
使用している Microsoft Visual SourceSafe についての情報が表示されます。この情報には、バージョン番号、著作権、法律およびライセンスに関する注意が含まれます。
  すべての Visual SourceSafe ユーザーがこのコマンドを使用できます。 

Add
Visual SourceSafe データベースに新しいファイルを追加します。
ss Add <local files> [-B] [-C] [-D-] [-H] [-I-] [-K] [-N] [-O] [-R] [-W] [-Y] [-?]
  このコマンドを使用するには、追加/名前の変更/削除を実行するプロジェクト権限が必要です。

Branch
共有リンクを切断し、プロジェクト内のファイルの分岐を作成します。
ss Branch <file name> [-C] [-H] [-I-] [-N] [-O] [-R] [-Y] [-?]
  このコマンドを使用するには、プロジェクトの追加、名前変更、および削除を実行する権限が必要です。
