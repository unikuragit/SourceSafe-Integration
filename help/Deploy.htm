Visual SourceSafe
Deploy (コマンド ライン)
参照 
 






プロジェクトを、Deploy_Path 初期化変数で指定したサーバーまたは FTP の位置に配置します。コマンド オプションの定義については、このヘルプ システムの「コマンド オプション」を参照してください。

構文

 コードのコピー
ss Deploy <project path> [-O] [-R] [-V]

プロジェクト権限

このコマンドを使用するには、プロジェクトの破棄を実行する権限が必要です。

解説

このコマンドは [最新バージョンの取得] コマンドによく似ていますが、このコマンドでは、ファイルの配置先が作業フォルダに限定されず、また既存のファイルを上書きする以外の操作ができません。プロジェクトに対して再帰的に使用すると (-R オプション)、Visual SourceSafe は、プロジェクトとサブプロジェクトのすべてのファイルを配置し、同じ名前のサブフォルダに格納します。

既定では、Deploy を使用すると、指定したプロジェクトの最新のバージョンがコピーされます。ただし、-V オプションを使用して旧バージョンを配置することもできます。旧バージョンのプロジェクトを配置する場合、プロジェクト内のファイルごとにバージョンを指定して取得できます。たとえば、Deploy コマンドを使用して、プロジェクト $/WORD のバージョン 7 を取得する場合、Test.html のバージョン 3、Help.htm のバージョン 10、Test.map のバージョン 5 を取得できます。

メモ

このコマンドでは、-V オプションでワイルドカードを使用して、たとえば ss Deploy -V* のように指定することはできません。

例

Web プロジェクト ($/SSAFE) を配置します。

 コードのコピー
ss Deploy $/SSAFE
